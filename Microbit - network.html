Jak to funguje

Micro:bity mohou komunikovat přes rádiový signál (2,4 GHz). Jeden micro:bit bude server (přijímá a zobrazuje data), ostatní budou klienti (měří a posílají).

Klíčové pojmy

radio.setGroup(číslo) Nastaví „kanál" sítě (0–255). Všechny micro:bity ve skupině musí mít stejné číslo.
radio.sendNumber(n) Pošle číslo všem micro:bitům na stejné skupině (broadcast).
radio.onReceivedNumber(fn) Handler — spustí se, když přijde číslo. Parametr receivedNumber.
radio.sendValue(klíč, hodnota) Pošle dvojici text+číslo. Server pozná, co data znamenají.
radio.setTransmitPower(1–7) Síla signálu. Nižší = menší dosah, méně kolizí.
serial number Unikátní číslo každého micro:bitu — lze použít jako „adresu" odesílatele.
Kolize Dva klienti pošlou zprávu ve stejný okamžik → jeden paket se ztratí.
Ztráta paketu Zpráva nedorazí (rušení, vzdálenost). Základní radio ji nepošle znovu automaticky.

SERVER — přijímač dat

Server naslouchá a zobrazuje přijatá čísla na displeji.

Blokový kód (MakeCode)

  1. Nastav skupinu

    Blok při spuštění → sekce Rádio → nastavit skupinu radia na 1

    Na displeji zobraz S1 (server), ať víš, které zařízení je server.

  2. Přijmi číslo

    Přidej blok při přijetí čísla přes rádio. Parametr se jmenuje recivedNum.

    Uvnitř zobraz přijaté číslo: zobrazit číslo → recivedNum

  3. (Rozšíření) Zobraz i sílu signálu

    Ve stejném handleru přidej: zobrazit číslo → síla signálu radia

JavaScript / Python ekvivalent

radio.setGroup(1)           // při spuštění
basic.showString("S")

radio.onReceivedNumber(function(receivedNumber) {
    basic.showNumber(receivedNumber)
})
Rozšíření: přijímej hodnoty s klíčem (sendValue) 
radio.onReceivedValue(function(name, value) {
    basic.showString(name)
    basic.showNumber(value)
})

Server teď zobrazí např. T a pak číslo teploty — víš, co data znamenají.

📡 KLIENT — odesílač dat

Klient měří senzorem a pravidelně posílá data na server.

  1. Nastav skupinu (stejnou jako server!)

    Blok při spuštění → nastavit skupinu radia na 1

    Zobraz na displeji písmeno nebo číslo, ať víš které zařízení je které.

  2. Posílej opakovaně

    Blok opakovat stále → přidej odeslat číslo → teplota (°C)

    Přidej pauzu: čekat 2000 ms (2 sekundy)

    Proč pauza? Co se stane, když tam pauza nebude?

  3. (Rozšíření) Pošli s klíčem

    Místo odeslat číslo použij odeslat hodnotu "teplota" = teplota (°C)

JavaScript ekvivalent

radio.setGroup(1)
basic.showString("A")    // identifikátor tohoto klienta

basic.forever(function() {
    let teplota = input.temperature()
    radio.sendNumber(teplota)   // nebo sendValue("t", teplota)
    basic.pause(2000)
})

Zadání

Vyber si úroveň podle toho, co ti teď připadá zvládnutelné.

Základní

Funkční síť

  • Server přijímá a zobrazuje čísla od min. 2 klientů
  • Klienti posílají teplotu každé 2 s
  • Správně nastavená skupina (neplete se s ostatními)
  • Na serveru vidím, že přicházejí data
Hotovo? Zkus záměrně způsobit kolizi — odstraň pauzu u obou klientů.
Rozšíření

Identifikace klientů

  • Použij sendValue místo sendNumber
  • Každý klient posílá i své ID (písmeno nebo číslo)
  • Server zobrazí ID a pak hodnotu
  • Přidej druhý senzor (světlo nebo akcelerometr)
  • Zaznamenej, kolik zpráv za minutu dorazí
Výzva

Detekce kolizí & protokol

  • Klient posílá čítač paketů (n++)
  • Server detekuje chybějící čísla → výpočet % ztráty
  • Náhodné zpoždění pro snížení kolizí
  • Navrhni vlastní „protokol" pro adresování konkrétního klienta
  • Porovnej ztrátu paketů na různé vzdálenosti
Dokumentuj! Zapiš si: Číslo skupiny · Jaký senzor · Co jsi zjistil o kolizích. Výsledky budeme porovnávat.

Jak změřit ztrátu paketů?

Klient posílá čítač (0, 1, 2, 3…). Server sleduje, jestli čísla jdou za sebou. Chybějící číslo = ztracený paket. Tohle je základ monitorování síťové kvality! 

// Klient — přidej čítač
let pocitadlo = 0
basic.forever(function() {
    radio.sendValue("n", pocitadlo)
    pocitadlo++
    basic.pause(1000)
})
